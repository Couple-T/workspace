# Image generation (graphic-designer asset pipeline)

The single reference for how the **graphic-designer** (`Fiona`) generates visual
assets — mascots, icons, logos, decorators, banners — during the design / PRD
pipeline, and how to turn that capability on in a freshly `aiworks sync`'d
workspace. If this is not set up, the design phase must **fail loud**, never
silently ship placeholder art marked dev-ready.

## The backend: `mcp-image`

Assets are generated by the **`mcp-image`** MCP server (npm package, run via
`npx -y mcp-image`), which exposes `mcp__mcp-image__generate_image` (Gemini —
"Nano Banana"). The graphic-designer drives it through the **`/image-generation`**
skill (`.claude/skills/image-generation/`), which structures prompts as
Subject–Context–Style.

- **Server declaration** — committed in `.mcp.json` (`mcp-image`, `stdio`,
  `npx -y mcp-image`), enabled in `.claude/settings.json` `enabledMcpjsonServers`.
- **Required secret** — `GEMINI_API_KEY` (get one at
  <https://aistudio.google.com/apikey>). `.mcp.json` reads it as
  `${GEMINI_API_KEY}` from the environment.
- **Output** — `IMAGE_OUTPUT_DIR` (defaults to `/tmp/aiworks-generated-images`);
  generated files are uploaded into the product's Figma **Assets page**, so the
  local dir is just scratch staging.
- **Quality** — `IMAGE_QUALITY=balanced` by default (presets: `fast` / `balanced`
  / `quality`). Budget rules for the team: `balanced` + `1K`, **≤2 generations per
  request**.

## Setup (one-time, per machine)

The key is kept **out of version control** in the git-ignored
`.claude/settings.local.json` `env` block. Claude Code exports that `env` to MCP
servers, where `.mcp.json`'s `${GEMINI_API_KEY}` expands it.

1. `aiworks sync` prepares `.claude/settings.local.json` — enables `mcp-image` and
   seeds an empty `GEMINI_API_KEY` placeholder. (A non-empty key is never
   clobbered; other settings are preserved.)
2. Edit `.claude/settings.local.json` and set the key:
   ```json
   {
     "enabledMcpjsonServers": ["mcp-image", "sonarqube", "postgres", "redis"],
     "env": { "GEMINI_API_KEY": "AIza…your-key…" }
   }
   ```
   (Alternatively, `export GEMINI_API_KEY=…` in the shell that launches Claude.)
3. Restart the Claude Code session so `mcp-image` starts with the key.

### Verify
```bash
claude mcp list        # mcp-image → ✔ Connected
```
A connected server with **no key** still cannot generate — confirm the key is set
too (`[ -n "$GEMINI_API_KEY" ]`, or a non-empty value in `settings.local.json`).

## How the pipeline fails loud (no silent placeholders)

| Layer | Behavior |
|---|---|
| `/prd` preflight (step 0b) | Before the design phase, detects whether `mcp-image` is connected **and** `GEMINI_API_KEY` is set. If not, warns the user and offers: **(a)** enable it, **(b)** proceed placeholder/specs-only with an explicit note, **(c)** abort. Never defaults into (b) silently. |
| graphic-designer | Availability gate first. If the tool is absent or a call errors on auth/key/quota, returns every asset as **`unavailable`** with a reason + fix — never improvises a placeholder it then calls `created`. Each asset carries a `status`: `created` / `reused` / `placeholder` / `unavailable`. |
| ux-ui-designer | A state depending on a `placeholder`/`unavailable` asset is **not dev-ready**: it goes in `asset_gaps` and forces `dev_ready:false`. |
| PRD report | Lists any placeholder/unavailable assets and `asset_gaps` as **not dev-ready (placeholder art)**, and restates how to enable real generation. |

## Cost

`mcp-image` calls the Gemini image API and bills to your `GEMINI_API_KEY`. The
budget rules above (`balanced`/`1K`, ≤2 generations per request) keep spend low;
raise quality only when a request explicitly demands it.
